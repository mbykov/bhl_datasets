Вот краткий и точный алгоритм работы нашего пайплайна
«Skeletal Assembly + LLM Paraphrase»:
1. Подготовка (Data Loading)

    Скрипт загружает три справочника: Variables (символы LaTeX + их названия на двух языках), Math Rules (математические операторы и шаблоны формул) и Templates (лингвистические заготовки).

2. Сборка «Скелета» (Python Logic) — Гарантия точности

    Выбор правила: Случайно выбирается математическая операция (например, fraction).
    Выбор переменных: Из справочника берутся 2 или 3 случайных объекта переменных (например, alpha и x). Это гарантирует, что имя «альфа» и символ \alpha всегда связаны.
    Генерация LaTeX: Python программно подставляет символы в шаблон (например, \frac{\alpha}{x}). LLM на этом этапе не участвует, что исключает ошибки в синтаксисе LaTeX.
    Выбор шаблона: Случайно выбирается текстовая заготовка из templates.json (например, {noun} {v1} на {v2}).

3. Очеловечивание (LLM Linguistic Layer) — Разнообразие

    Формирование промпта: Скрипт передает модели Qwen-7B:
        Сырую заготовку («дробь альфа на икс»).
        Случайный стиль (Академический, Разговорный, Вопрос и т.д.).
        Строгое правило: «Не менять имена переменных и не использовать символы».
    Генерация текста: Модель перефразирует заготовку в живую русскую и английскую речь, адаптированную под микрофон (STT).

4. Валидация и Очистка (Post-processing)

    Фильтрация: Скрипт принудительно вырезает из текста любые случайно оставшиеся символы ($, \, {).
    Синхронизация: Скрипт собирает финальный JSON, объединяя гарантированный LaTeX от Python и живой текст от LLM.
    Проверка: Если в ответе пустые поля или критические ошибки, пример отбрасывается.

5. Сохранение (Checkpointing)

    Каждые 100 успешных примеров пачка данных дописывается в файл final_math_dataset.json.
    Оперативная память очищается, чтобы избежать зависаний при генерации 100 000+ строк.

Результат: Математически безупречный, лингвистически разнообразный датасет, готовый для обучения нейросети.
