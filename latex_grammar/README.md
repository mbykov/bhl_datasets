# Math-to-LaTeX Grammar Factory 🏭

## Описание проекта
Этот проект представляет собой комбинаторную фабрику данных для обучения малых языковых моделей (LLM, например Qwen2.5-1.5B) переводу естественной математической диктовки в профессиональный код LaTeX.
В отличие от простых генераторов, данный инструмент имитирует живую человеческую речь, учитывая вариативность синонимов, падежное управление и вложенность операций.

## Алгоритм генерации (Архитектура)
Генерация строится на четырехуровневой модели данных, разнесенной по JSON-файлам:

### Слой Атомов (atoms.json): Математические «кирпичики». Каждый атом имеет свой LaTeX-шаблон, тип (инфиксный, префиксный, функциональный) и привязку к разделу математики (sec).
 - Слой Словаря (dictionary.json): Лингвистическая оболочка. Для каждого атома прописаны:
   - Nouns: Существительные («сумма», «корень»).
   - Verbs: Глаголы прямого действия («извлечь», «вычислить»).
   - Verbs_directed: Направленные действия («прибавить А к Б»).
   - Short: Краткие связки («плюс», «минус», «на»), используемые в сложных цепочках.
 - Слой Переменных (variables.json): Огромный набор латинских и греческих букв, векторов и констант для визуального разнообразия.
 - Слой Шаблонов (templates.json): Речевые скелеты, которые задают порядок слов в фразе.

### Стратегия сборки датасета (produce_structured)
Скрипт factory.py не просто выбирает случайные строки, а проходит через системный цикл для каждого из 12 разделов математики:

 - Этап 1: Базовая грамотность (n_base)
    Скрипт перебирает все доступные синонимы каждого термина. Это гарантирует, что модель выучит базу: «сложить», «плюс» и «сумма» — это всё один и тот же символ +.
 - Этап 2: Сложная вложенность (n_nested)
    Используется рекурсивная генерация. Операция может стать аргументом другой операции (например, «корень из суммы»). Здесь модель учится понимать приоритеты и правильно расставлять скобки.
 - Этап 3: Уравнения (Тип 1 и 2)
    Генерируются конструкции со знаком равенства.
   - Тип 1: Сложное выражение равно числу/переменной.
   - Тип 2: Сложное выражение равно другому сложному выражению (учит модель понимать баланс формулы).

## Как пользоваться

 - Наполнение: Добавьте новые термины в data/atoms.json и их описание в data/dictionary.json.
 - Настройка: В блоке if __name__ == "__main__": в файле factory.py задайте желаемый объем генерации (N_NESTED, N_EQ).
 - Запуск:
```bash
    python3 factory.py
```
    Скрипт автоматически найдет все секции и допишет новые примеры в results/dataset.jsonl, сохраняя историю предыдущих запусков.
